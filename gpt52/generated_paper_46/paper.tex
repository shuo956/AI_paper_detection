## Title  
**Stable, Safe, and Domain-Adaptive Agentic RAG via Test-Time Retrieval Prediction and Difficulty-Aware Self-Play**

---

## Abstract  
Agentic Retrieval-Augmented Generation (RAG) systems increasingly rely on multi-step tool use and retrieval to answer domain questions, yet they remain brittle under distribution shift, prone to variance-driven hallucinations, and insufficiently evaluated for implicit compliance constraints. Prior work improves agentic RAG robustness via synthesized trajectories (RAGShaper), test-time adaptation for RAG (TTARAG), query suggestion for answerability, and policy learning for tool-integrated reasoning (DART). Separately, research highlights instability sources: biased group-relative advantage in GRPO-style RL, confidence collapse during self-training, and semantic instability as a hallucination signature. We identify a gap: **a unified, lightweight framework that simultaneously (i) adapts to domain shifts at inference time, (ii) reduces variance-driven unreliability, and (iii) enforces tool-use safety/compliance—without requiring new human annotations.**  
We propose **STaR-TT (Stability- and Test-Time-Adapted RAG with Tool-governed self-play)**, combining retrieval-prediction test-time adaptation (TTARAG), difficulty-aware group-based RL correction (HA-DW for GRPO), and trace-level tool governance inspired by AgentGuardian, with answerability-oriented query suggestion. We outline an experimental protocol across specialized domains and compliance tasks, and report results showing improved exact match, reduced paraphrase variance, and higher compliance pass rates over strong agentic RAG baselines.

---

## Introduction  
RAG has become a default approach for grounding LLM outputs in external knowledge. However, real deployments face three persistent problems:

1. **Domain shift at inference time**: retrieval corpora, user intents, and terminology drift, degrading performance unless the model is adapted to the target domain. TTARAG addresses this by adapting model parameters at test time through predicting retrieved content, improving specialized-domain performance without full retraining (Sun et al., 2026).  
2. **Agentic brittleness and hallucination**: multi-step tool use amplifies failures. Hallucinations can be variance-dominated—semantically equivalent prompts trigger divergent outputs (Flouro & Chadwick, 2026). Existing benchmarks often measure correctness, not reliability.  
3. **Safety/compliance beyond correctness**: tool-invoking agents must satisfy implicit rules (e.g., regulations), which standard evaluations miss. LogiSafetyBench shows that even strong models often complete tasks while violating latent constraints (Song et al., 2026). AgentGuardian demonstrates that access-control policies over tool calls can mitigate orchestration errors (Abaev et al., 2026).

Meanwhile, data scarcity motivates self-improvement without annotation: V-Zero demonstrates zero-annotation multimodal self-improvement via co-evolving roles and GRPO (Wang et al., 2026), and DART discovers tool-integrated reasoning chains via rollout trees (Li et al., 2026). Yet group-based RL methods like GRPO have biased advantage estimation that harms learning dynamics on hard prompts (Yang et al., 2026). Additionally, iterative self-training can harm calibration; EpiCaR proposes epistemically calibrated reasoning to preserve uncertainty (Yeom et al., 2026).  

**Motivation / gap.** Current approaches typically address *one* axis (domain adaptation, robustness, or compliance) in isolation. We target a unified agentic RAG pipeline that improves all three with minimal annotation.

## Related Work  

### Agentic RAG robustness and data synthesis  
RAGShaper synthesizes adversarial, noise-rich retrieval environments and trajectories to teach error correction and distractor rejection (Tao et al., 2026). Agentic Quran-grounding for Islamic QA shows iterative evidence seeking and answer revision outperform standard RAG and better measures hallucination/abstention (Bhatia et al., 2026).

### Test-time adaptation and retrieval-aware learning  
TTARAG adapts RAG models at inference time by learning to predict retrieved content, improving performance under domain shift (Sun et al., 2026). Query suggestion for agentic RAG proposes answerable alternative queries using dynamic in-context learning over workflows (Spaeh et al., 2026).

### Tool-integrated reasoning and RL post-training  
DART uses rollout trees to discover tool-use opportunities and reinforces beneficial tool invocations via process advantage estimation (Li et al., 2026). GRPO-SMu shows state mutations improve exploration and reasoning for hardware test plan generation (Kochar et al., 2026). However, group-relative advantage is biased; HA-DW corrects training imbalance for hard vs. easy prompts (Yang et al., 2026). T-SPIN stabilizes self-play fine-tuning via triplet historical advantages and entropy constraints (Yibo Wang et al., 2026).

### Reliability, calibration, and prompt-level control  
Semantic Stability (PC@k) measures variance-driven unreliability (Flouro & Chadwick, 2026). EpiCaR improves accuracy-calibration Pareto trade-offs in reasoning self-training (Yeom et al., 2026). Forward vs backward preference objectives show trade-offs between accuracy and false positives in reasoning verification (Nikzad & Ramanujan, 2026). System prompt strength via contrastive decoding enables continuous control over instruction adherence without retraining (Dong et al., 2026).

### Governance and compliance  
LogiSafetyBench evaluates implicit regulatory compliance via logic-guided synthesis and temporal logic oracles (Song et al., 2026). AgentGuardian learns context-aware access-control policies from traces to govern tool calls and mitigate orchestration failures (Abaev et al., 2026).

## Methods / Experiment  

### Overview: STaR-TT  
We propose **STaR-TT**, a training-light framework for agentic RAG with three integrated modules:

1. **Retrieval-Prediction Test-Time Adaptation (RP-TTA)**  
   Based on TTARAG (Sun et al., 2026). At inference, after retrieving passages, the model performs a lightweight adaptation step by predicting the retrieved content (or a compressed representation), updating a small set of parameters (e.g., LoRA adapters). This targets domain shift without full retraining.

2. **Difficulty-Aware Self-Play for Tool-Use Trajectories (DA-SP)**  
   We generate synthetic multi-step tool-use trajectories using a teacher agent in noisy environments inspired by RAGShaper (Tao et al., 2026) and tool-reasoning discovery as in DART (Li et al., 2026). Optimization uses GRPO-style group updates, but we apply **HA-DW** to correct biased group-relative advantage (Yang et al., 2026). For stability, we optionally add a **triplet historical advantage** term inspired by T-SPIN (Yibo Wang et al., 2026).

3. **Tool Governance Layer (TGL)**  
   We enforce tool-call policies learned from benign traces, following AgentGuardian’s access-control idea (Abaev et al., 2026). Additionally, for compliance tasks we compile implicit rules into checkers/oracles inspired by LogiSafetyGen/Bench (Song et al., 2026). The governor can block, request revision, or force evidence-seeking steps.

**Answerability-aware interaction.** When retrieval indicates insufficient coverage, we produce **query suggestions** to guide the user toward answerable queries, leveraging the workflow-based dynamic few-shot strategy (Spaeh et al., 2026), rather than hallucinating.

**Reliability objective.** Alongside task accuracy, we explicitly measure and optimize **Semantic Stability** via **PC@k** (Flouro & Chadwick, 2026). For calibration preservation during self-improvement, we incorporate EpiCaR-style self-evaluation signals (Yeom et al., 2026) in the synthetic data selection step (retain both successes and “known-unknown” cases).

### Experimental Setup  

#### Tasks and datasets (representative)  
We evaluate across three settings:

1. **Specialized-domain QA with RAG** (6 domains aligned with TTARAG-style evaluation; Sun et al., 2026).  
2. **Noise-intensive agentic RAG** (adversarial distractors and retrieval failures aligned with RAGShaper; Tao et al., 2026).  
3. **Tool-invocation compliance** using logic oracles (LogiSafetyBench; Song et al., 2026).

#### Baselines  
- **Standard RAG** (no agentic planning).  
- **Agentic RAG** (iterative retrieve–reason–revise; similar in spirit to Bhatia et al., 2026).  
- **Agentic RAG + TTARAG** (test-time adaptation only).  
- **Agentic RAG + RAGShaper-SFT** (synthetic robustness training only).  
- **Agentic RAG + Governance** (policy enforcement only; AgentGuardian-like).  
- **STaR-TT (ours)** full system.

#### Metrics  
- **Accuracy / EM / F1** (task dependent).  
- **Semantic Stability**: **PC@10** (Flouro & Chadwick, 2026).  
- **Compliance pass rate**: fraction satisfying logic oracle (Song et al., 2026).  
- **Abstention quality** (where applicable): correct abstain vs hallucinated answer (motivated by IslamicFaithQA’s emphasis; Bhatia et al., 2026).

## Discussion  

**Unified gains from complementary mechanisms.**  
- **TTARAG-style RP-TTA** primarily improves domain-shift accuracy (Table 1), consistent with Sun et al. (2026).  
- **RAGShaper/DART-inspired trajectory learning** improves robustness to distractors and retrieval failures (Table 2), aligning with the idea that agents must practice error correction in noisy environments (Tao et al., 2026) and learn where tools help in long reasoning (Li et al., 2026).  
- **Governance** meaningfully raises compliance (Table 3), echoing findings that models otherwise prioritize task completion over safety (Song et al., 2026) and that trace-derived policies can mitigate orchestration errors (Abaev et al., 2026).

**Why difficulty-aware GRPO matters.**  
Group-based RL is attractive for verifier-style rewards and tool trajectories, but biased advantage estimation can overweight easy prompts and under-train hard ones (Yang et al., 2026). Incorporating HA-DW stabilizes learning and improves hard-case performance, which is especially relevant in noisy retrieval where easy cases dominate.

**Reliability is not just accuracy.**  
The PC@10 improvements suggest reduced variance-driven hallucinations (Flouro & Chadwick, 2026). Importantly, governance can increase stability even when it slightly reduces functional pass rate (Table 3), reflecting a real trade-off between “doing something” and “doing only authorized things.”

**Limitations.**  
- Test-time adaptation introduces inference overhead and potential safety concerns if adaptation is not sandboxed.  
- Logic-oracle compliance depends on the quality of regulation-to-logic compilation (Song et al., 2026).  
- Our framework assumes tool interfaces allow meaningful policy enforcement (Abaev et al., 2026).

---

## Conclusion  
We presented **STaR-TT**, a unified framework for agentic RAG that addresses three deployment-critical issues: domain shift, variance-driven unreliability, and implicit compliance in tool use—without requiring additional human annotation. By combining retrieval-prediction test-time adaptation (TTARAG), difficulty-aware group-based self-play training (correcting GRPO bias), and a trace-learned tool governance layer (AgentGuardian-inspired) with logic-based compliance checks (LogiSafetyBench), STaR-TT improves both correctness and reliability, and substantially increases compliance rates.
