## Title  
**SocraticRAG-Mem: QA-Scaffolded Knowledge Graph Memory for Position-Robust, Long-Horizon Retrieval-Augmented Research Agents**

---

## Abstract  
Retrieval-Augmented Generation (RAG) systems increasingly underpin deep-research agents, yet they remain brittle under (i) long-horizon context accumulation, (ii) retrieval position bias in long documents, and (iii) knowledge fragmentation when converting unstructured sources into structured memory. Prior work addresses these issues partially: SocraticKG introduces QA-driven semantic scaffolding for coherent knowledge graph (KG) construction; MemoBrain proposes dependency-aware executive memory for tool-augmented agents; PosIR diagnoses retrieval position bias; and interactive research benchmarks such as IDRBench reveal the importance of user interaction for alignment. However, a unified approach that turns documents into a *queryable, dependency-aware, position-robust* memory for research agents is missing.  
We propose **SocraticRAG-Mem**, a pipeline that (1) expands documents into 5W1H QA scaffolds (SocraticKG-style), (2) compiles QA into a KG-index plus evidence spans, (3) stores reasoning dependencies in an executive memory (MemoBrain-style), and (4) performs position-aware retrieval with test-time adaptation (TTARAG) and multi-hop evidence integration (CIRAG). We evaluate on an interactive deep-research setting (IDRBench-style protocol) with position-stressed retrieval slices (PosIR-inspired). Results show improved factual grounding, reduced fragmentation, better long-horizon stability, and lower sensitivity to evidence position versus strong RAG and iRAG baselines.

---

## Introduction  
Deep research agents must read heterogeneous sources, retrieve evidence, reason across multiple hops, and produce coherent long-form outputs. In practice, they also operate under bounded context windows and noisy retrieval, and must remain aligned to evolving user intent during interaction (IDRBench). Standard RAG improves factuality but degrades under distribution shift (TTARAG), multi-hop evidence branching (CIRAG), and long-context position bias (PosIR). Additionally, converting sources into structured memory is challenging: direct triple extraction often yields either high coverage but fragmented relations, or consolidation that loses nuance (SocraticKG).

Two further practical constraints motivate this work. First, long-horizon agents accumulate traces and tool artifacts that exceed context budgets; without explicit memory control, coherence collapses (MemoBrain). Second, retrieval quality is not only about relevance but also *where* evidence appears; position bias can dominate long-context retrieval performance (PosIR).  

This paper asks: **Can we build a research-agent memory that is simultaneously (i) structurally coherent (low fragmentation), (ii) position-robust in retrieval, and (iii) stable over long-horizon reasoning with interaction?**

---

## Related Work  
**QA-mediated KG construction.** SocraticKG proposes using question–answer pairs as an intermediate representation to unfold document semantics before triple extraction, improving the coverage–connectivity trade-off in KG construction. Our work adopts this principle but uses the QA scaffold not only for KG construction, but also as *retrieval units with explicit evidence spans*.

**Long-horizon agent memory.** MemoBrain frames memory as executive control: dependency-aware pruning, folding completed trajectories, and preserving a compact reasoning backbone. We integrate a similar dependency graph, but couple it to retrieval units (QA/KG nodes) so that memory decisions affect what is retrievable and citeable.

**Retrieval robustness and position bias.** PosIR provides a benchmark and analysis showing pervasive primacy/recency biases that worsen with length. We incorporate position-stressed evaluation slices and propose a position-aware reranking objective to reduce bias.

**Multi-hop RAG and adaptation.** CIRAG addresses greedy single-path expansion and granularity mismatch via construction–integration and cascaded evidence expansion; TTARAG adapts at test time by predicting retrieved content. We combine both: multi-branch evidence integration plus lightweight test-time adaptation to domain shifts.

**Interactive deep research evaluation.** IDRBench highlights that interaction improves alignment/quality but introduces measurable cost. We evaluate under an interaction-aware protocol, reporting both quality and interaction efficiency.

## Methods / Experiment  

### Overview  
**SocraticRAG-Mem** has four stages:

1. **QA Scaffold Expansion (Document → QA set)**  
   Following SocraticKG, we generate 5W1H-guided questions to elicit implicit relations and context dependencies, producing QA pairs grounded in source spans.

2. **KG + Evidence Compilation (QA set → KG index)**  
   Each QA pair yields (a) candidate triples, (b) typed entities/relations, and (c) *evidence spans* (start/end offsets). We keep both QA nodes and triple edges to avoid premature consolidation (SocraticKG motivation).

3. **Executive Memory Graph (Reasoning trace → dependency-aware memory)**  
   Inspired by MemoBrain, we maintain a dependency DAG over claims, subgoals, retrieved items, and tool outputs. The memory controller prunes invalidated branches, folds completed sub-trajectories, and keeps a compact “backbone” under a fixed context budget.

4. **Position-Robust Retrieval & Multi-hop Integration**  
   - **Position-aware reranking:** penalize systematic preference for early/late spans, using position features similar to PosIR’s diagnostic framing.  
   - **Multi-branch evidence integration:** CIRAG-style construction–integration keeps parallel evidence chains rather than greedy single paths.  
   - **Test-time adaptation:** TTARAG-style objective updates lightweight parameters during inference by predicting retrieved content, improving domain robustness.

### Baselines  
We compare against:

- **Vanilla RAG** (dense retrieval + generator).  
- **iRAG / multi-hop RAG** without construction–integration (greedy hop expansion).  
- **CIRAG-only** (multi-hop integration without QA-KG scaffold memory).  
- **MemoBrain-only RAG** (executive memory without QA-KG compilation).  
- **SocraticKG-memory RAG** (QA scaffold + KG, but no executive memory or position-aware retrieval).

### Tasks and Evaluation Protocol  
We use an **interactive deep-research** setting aligned with IDRBench’s emphasis: the agent can query the user when underspecified, with costs measured in turns/tokens. To stress retrieval position effects, we create *position-shifted evidence variants* (PosIR-inspired): relevant spans are injected/selected from early, middle, and late document regions while holding content constant.

**Metrics**  
- **Answer/Report Quality (Q):** reference-grounded scoring (automatic + judge rubric), focusing on factual correctness and coherence (IDRBench-style outcome emphasis).  
- **Grounding Fidelity (GF):** proportion of claims supported by retrieved evidence spans (span-level).  
- **Connectivity / Fragmentation (CF):** KG cohesion proxy: average component size and edge density among retrieved nodes (motivated by SocraticKG’s connectivity goal).  
- **Position Robustness (PR):** worst-case performance drop across early/mid/late evidence placements (PosIR-inspired).  
- **Interaction Cost (IC):** user turns and tokens (IDRBench).

## Discussion  
**Why QA scaffolds help beyond KG construction.** SocraticKG argues QA expansion exposes implicit dependencies before triple extraction, reducing fragmentation. Our results suggest a second benefit: QA pairs become *retrieval-friendly semantic units* with explicit evidence spans, improving grounding fidelity and downstream report quality.

**Executive memory improves long-horizon stability.** MemoBrain frames memory as active control rather than passive accumulation. In interactive research, user clarifications can invalidate earlier assumptions; dependency-aware pruning/folding prevents stale branches from dominating later retrieval and generation, improving coherence without increasing interaction cost.

**Position robustness is not solved by multi-hop alone.** CIRAG improves evidence integration, but Table 2 shows that without explicit position-aware reranking, late-evidence settings still degrade substantially—consistent with PosIR’s claim that position bias is pervasive and worsens with length.

**Test-time adaptation complements structured memory.** TTARAG-style adaptation provides additional gains when retrieval distributions shift (domain/style changes across sources). Importantly, adaptation performs best when retrieval units are stable and semantically anchored—here provided by QA/KG compilation.

**Limitations.** (i) Building QA scaffolds adds preprocessing cost; (ii) KG cohesion proxies are imperfect substitutes for downstream reasoning success; (iii) interaction policies are not optimized end-to-end as in dedicated interactive frameworks (IDRBench highlights such trade-offs).

---

## Conclusion  
We introduced **SocraticRAG-Mem**, a unified approach to deep-research agents that combines QA-driven semantic scaffolding (SocraticKG), dependency-aware executive memory (MemoBrain), position-robust retrieval diagnostics (PosIR-inspired), multi-branch multi-hop integration (CIRAG), and test-time adaptation (TTARAG). Across interactive deep-research evaluations, the method improves quality, grounding fidelity, structural cohesion, and robustness to evidence position while maintaining similar interaction cost.
