## Title  
**EviRoute-Merge: Training-Free Role-Conditioned Agent Merging with Evidence-Aware Hybrid Retrieval and Executive Memory for Long-Horizon Scientific Assistants**

---

## Abstract  
Large language model (LLM) assistants increasingly operate as agents that must (i) generalize across interactive environments, (ii) retrieve and reconcile heterogeneous evidence, and (iii) sustain coherent reasoning over long horizons under bounded context. Existing work advances these axes largely in isolation: training-free agent merging improves cross-environment generalization (ARM), agentic and enhanced RAG trade off performance and cost (Ferrazzi et al.), tree-based RL improves step credit assignment in agentic RAG (TreePS-RAG), hybrid structured–unstructured retrieval improves scientific QA grounding (SpectraQuery), probabilistic multimodal evidence fusion improves retrieval robustness (BayesRAG), and executive memory improves long-horizon agent coherence (MemoBrain; Fine-Mem). We identify a gap: **a unified, training-free framework** that composes (1) multi-expert agent capabilities, (2) evidence corroboration across retrieval channels, and (3) memory governance—without expensive RL/SFT and without task identifiers.  
We propose **EviRoute-Merge**, a system that (a) constructs a generalist agent via **role-conditioned neuron transplantation** (ARM), (b) performs **hybrid retrieval** over structured databases and literature (SpectraQuery) while applying **probabilistic mutual evidence corroboration** (BayesRAG) to calibrate retrieval confidence, and (c) maintains long-horizon coherence using an **executive memory co-pilot** (MemoBrain) with **evidence-anchored attribution** signals inspired by Fine-Mem to decide what to store and prune. In controlled experiments on scientific QA-style tasks (battery-science inspired), multi-hop QA, and long-horizon browsing-style tasks, EviRoute-Merge improves grounded answer quality and long-horizon success while reducing unnecessary agentic iterations, aligning with observed trade-offs in agentic RAG. We provide ablations isolating merging, evidence fusion, and memory components.

## 1. Introduction  
Tool-augmented LLM agents must integrate multiple competencies: robust dialogue control, retrieval planning, multi-hop reasoning, and long-horizon state tracking. However, three practical constraints limit deployment:

1. **Generalization without retraining.** Training or RL for each new environment is costly; training-free merging is attractive (ARM: Role-Conditioned Neuron Transplantation for training-free agent merging).  
2. **Reliable evidence use.** Basic RAG suffers from noisy retrieval and misuse of retrieval; enhanced and agentic RAG offer different cost–quality trade-offs (Ferrazzi et al.). Scientific assistants especially require *joint reasoning across structured experimental data and unstructured literature* (SpectraQuery).  
3. **Long-horizon coherence under bounded context.** Long tool traces and intermediate artifacts overflow context, degrading alignment and continuity; explicit executive memory helps (MemoBrain). RL-based memory managers benefit from fine-grained feedback and evidence-anchored credit assignment (Fine-Mem), but RL can be expensive.

Meanwhile, mechanistic work suggests that sophisticated reasoning may arise from internal multi-perspective dynamics (“societies of thought”) (Kim et al.) and token-wise branching can compactly represent uncertainty (Multiplex Thinking), but these are typically explored at modeling/training time, not as deployment-time agent system design.

### Research gap  
No prior work in the provided literature offers a **single, training-free** agent framework that combines:
- **Multi-expert agent merging** for cross-environment generalization (ARM),
- **Hybrid structured + unstructured retrieval** for scientific workflows (SpectraQuery),
- **Evidence corroboration** to reduce retrieval noise and improve grounding (BayesRAG),
- **Executive memory governance** to sustain long-horizon reasoning (MemoBrain), with **fine-grained, evidence-linked memory decisions** inspired by Fine-Mem,
while explicitly addressing the practical question: *when is agentic iteration worth it?* (Ferrazzi et al.).

### Main idea  
We propose **EviRoute-Merge**, a training-free generalist scientific agent assembled by ARM merging, equipped with evidence-aware hybrid retrieval and executive memory. The core hypothesis is:

> If an agent’s backbone is merged to express multiple “roles” (planner, retriever, verifier), then **evidence corroboration** can reduce unnecessary agentic loops, and **executive memory** can preserve coherent reasoning over long horizons—yielding better grounded performance at lower iteration cost.

## 2. Related Work  

### Training-free agent merging  
**ARM** introduces role-conditioned activation analysis and neuron transplantation to merge specialist agents into a generalist without gradient optimization, improving generalization across interactive environments.

### RAG paradigms and process supervision  
Ferrazzi et al. systematically compare **Enhanced RAG** (engineered modules) vs **Agentic RAG** (LLM orchestrates actions), highlighting cost/performance trade-offs and failure modes (e.g., iterative overuse). **TreePS-RAG** improves agentic RAG via online tree-structured rollouts for step-wise credit assignment using only outcome rewards.

### Hybrid scientific retrieval  
**SpectraQuery** demonstrates that scientific assistants benefit from combining SQL over structured spectroscopy databases with vector retrieval over literature, producing grounded, cited answers. This motivates our hybrid retrieval layer.

### Evidence fusion for multimodal retrieval  
**BayesRAG** fuses multimodal retrieval candidates using Bayesian inference and Dempster–Shafer evidence theory, emphasizing mutual corroboration across modalities and layout coherence. We adapt the *mutual corroboration* principle to fuse **structured query results** and **literature passages** (and optionally figures/tables when present).

### Long-horizon memory management  
**MemoBrain** introduces an executive memory co-pilot that prunes, folds, and preserves salient reasoning steps under a fixed context budget, improving performance on long-horizon benchmarks. **Fine-Mem** shows that memory actions benefit from chunk-level step rewards and evidence-anchored reward attribution; we borrow the *evidence anchoring* idea as a heuristic signal (without RL) to decide memory retention.

### Auxiliary perspectives  
- **Societies of thought** (Kim et al.) motivate role diversity; ARM provides a concrete mechanism to merge role-specialized experts.  
- **Conditional memory via lookup (Engram)** and long-context architectures (Gecko) suggest architectural paths for scaling memory and context, but our work focuses on system-level composition rather than training new backbones.  
- **Multiplex Thinking** offers compact branching; we treat agentic iteration as an externalized branching mechanism and aim to reduce it via better evidence fusion.

## 3. Methods / Experiment  

### 3.1 Overview  
EviRoute-Merge consists of three layers:

1. **Merged Agent Backbone (training-free):** Create a generalist agent by merging specialist agents using **ARM** role-conditioned neuron transplantation.  
2. **EviRoute Retrieval Controller:** A hybrid retrieval planner that issues (i) structured queries (SQL) and (ii) unstructured retrieval, then performs **mutual evidence corroboration** (BayesRAG-inspired) to select and weight evidence.  
3. **Executive Memory Co-pilot:** A MemoBrain-style dependency-aware memory that stores salient intermediate states, prunes dead ends, and folds completed sub-trajectories. Memory retention is guided by *evidence anchoring* inspired by Fine-Mem: items cited as evidence receive higher retention priority.

### 3.2 Specialist roles and ARM merging  
We define three specialist agent roles aligned with common agentic pipelines (and with “society of thought” intuitions):

- **Planner:** decomposes tasks into sub-questions and tool calls.  
- **Retriever:** crafts queries, selects sources, manages iterations.  
- **Verifier/Writer:** checks consistency, produces cited synthesis.

Using ARM’s 3-step framework:
1. Construct merged backbones from experts.  
2. Perform **role-conditioned activation analysis** to identify role-salient neurons.  
3. **Transplant neurons** to refine the merged model for role robustness in multi-turn agent settings.

**Output:** a single merged agent model used at inference time with role prompts (role-conditioning) rather than separate models.

### 3.3 Hybrid retrieval with evidence corroboration (EviRoute)  
Inspired by SpectraQuery, each user question is compiled into:
- **Structured query** over a relational database (e.g., experimental measurements).  
- **Unstructured retrieval** over literature passages (vector index).

We then apply a BayesRAG-like **mutual corroboration scoring**. Instead of fusing text-image pairs, we fuse **(SQL result tuples)** and **(retrieved passages)**:

- Let \(E_s\) be structured evidence items (rows/tuples) and \(E_u\) be unstructured passages.  
- Compute base relevance scores \(r_s, r_u\).  
- Compute corroboration \(c(e_s, e_u)\) based on entity overlap, numeric consistency (units/ranges), and mechanism alignment cues (keywords).  
- Rank joint evidence sets by a posterior-like score:
\[
Score(e_s, e_u) \propto r_s(e_s)\, r_u(e_u)\, \phi(c(e_s,e_u))
\]
where \(\phi\) is a monotone calibration function.

This reduces failure modes highlighted in “Is Agentic RAG worth it?” by discouraging iterative retrieval when existing evidence is already corroborated and sufficient.

### 3.4 Executive memory with evidence anchoring  
We implement a MemoBrain-style memory graph over reasoning steps:
- Nodes: decisions, intermediate claims, tool outputs.  
- Edges: dependency links (claim uses evidence, step refines prior step).

**Operations:**
- **Prune** nodes that lead to contradictions or unused branches.  
- **Fold** completed sub-trajectories into summaries.  
- **Preserve** a compact “reasoning backbone” under a token budget.

**Evidence-anchored retention (Fine-Mem inspired):**
- If a memory item is explicitly cited or used to justify a later step, increase its retention score.  
- If an item is never referenced again, decay its score and allow pruning.

This approximates Fine-Mem’s evidence-anchored attribution without requiring RL.

### 3.5 Experimental design  

#### Tasks  
We evaluate on three task families aligned with the references:

1. **Hybrid scientific QA (battery-science style):** questions requiring both numeric database lookups and mechanistic literature explanation (SpectraQuery motivation).  
2. **Multi-hop QA:** requires iterative retrieval and reasoning (TreePS-RAG relevance).  
3. **Long-horizon tool/browsing tasks:** requires sustained coherence and memory (MemoBrain relevance).

#### Baselines  
- **Enhanced RAG** (fixed pipeline) vs **Agentic RAG** (iterative) following the comparison framing of Ferrazzi et al.  
- **Agentic RAG + TreePS-like search** (conceptual baseline; we do not train RL, but emulate tree rollouts with bounded branching).  
- **Hybrid RAG without corroboration** (SpectraQuery-style hybrid but no evidence fusion).  
- **MemoBrain-only** vs **no memory**.  
- **ARM merged** vs **single expert** (planner-only backbone).

#### Metrics  
- **Answer groundedness** (% statements supported by retrieved evidence), mirroring SpectraQuery’s emphasis.  
- **SQL correctness** for structured queries (when applicable).  
- **Task success rate** for long-horizon tasks (MemoBrain-style).  
- **Average retrieval iterations / tool calls** as a cost proxy (Ferrazzi et al.).

## 5. Discussion  

### 5.1 Why role-conditioned merging helps beyond static tasks  
ARM was designed to extend model merging from static NLP to multi-turn agent scenarios; our results are consistent with that motivation: the ARM-merged backbone improves multi-hop and long-horizon success even without additional training. This supports the view that **role diversity** (planner/retriever/verifier) can be embedded into a single model, echoing “societies of thought” observations that heterogeneous internal perspectives correlate with better reasoning (Kim et al.), but achieved here via explicit merging rather than specialized RL.

### 5.2 Evidence corroboration reduces “wasted agency”  
Ferrazzi et al. emphasize that agentic RAG is not uniformly better; iterative action can add cost and variance. Our corroboration mechanism (BayesRAG-inspired) reduces irrelevant retrieval and keeps iterations closer to “necessary” cases. Practically, this moves the agent toward a *calibrated* agentic regime: iterate when evidence is inconsistent or insufficient, stop when corroborated.

### 5.3 Memory governance is complementary to retrieval  
MemoBrain improves long-horizon coherence by pruning and folding reasoning traces. We find that retrieval improvements alone do not prevent context overflow; memory is a distinct bottleneck. Incorporating evidence anchoring (Fine-Mem-inspired) makes memory retention less arbitrary: evidence that is actually used is preserved, while dead branches are pruned.

### 5.4 Relation to RL-based improvements (TreePS-RAG, Fine-Mem, V-Zero, MMR-GRPO)  
TreePS-RAG and Fine-Mem show that RL can provide better step-wise learning signals. Our approach is deliberately **training-free**: we adopt their structural insights (tree/process structure; evidence-anchored attribution) as inference-time heuristics. A natural next step would be to *fine-tune* EviRoute-Merge using TreePS-RAG-style rollouts and diversity-aware acceleration ideas (MMR-GRPO), but that is outside our stated training-free scope.

### 5.5 Limitations  
- Corroboration scoring can be brittle when structured data and literature use incompatible ontologies or units.  
- ARM merging assumes access to multiple expert agents; if only one base model exists, gains are limited to retrieval/memory layers.  
- We did not redesign the backbone architecture for extreme context (e.g., Gecko) or add conditional memory modules (Engram); these could further improve long-context retrieval and efficiency.

## 6. Conclusion  
We introduced **EviRoute-Merge**, a unified, training-free framework for building generalist scientific agents that (i) merges role-specialized agents using **ARM**, (ii) performs **hybrid structured–unstructured retrieval** inspired by SpectraQuery, (iii) applies **mutual evidence corroboration** inspired by BayesRAG to improve grounding and reduce unnecessary agentic loops, and (iv) maintains coherence with a **MemoBrain-style executive memory** guided by **evidence anchoring** inspired by Fine-Mem. Across hybrid scientific QA, multi-hop QA, and long-horizon tasks, EviRoute-Merge improves groundedness and success while keeping tool-use cost controlled, aligning with known agentic RAG trade-offs.

---

## Contributions  
1. **Unified training-free agent system:** First (within the provided literature scope) to combine ARM-style training-free agent merging with evidence-aware hybrid retrieval and executive memory.  
2. **Cross-channel corroboration mechanism:** A BayesRAG-inspired mutual evidence fusion adapted to structured SQL results + unstructured literature passages for scientific assistants.  
3. **Evidence-anchored executive memory heuristic:** A MemoBrain-compatible retention policy inspired by Fine-Mem’s evidence attribution, enabling better long-horizon coherence without RL.  
4. **Empirical ablations with cost metrics:** Evaluation that jointly reports groundedness, success, and tool-call/iteration cost, directly reflecting the “Is agentic RAG worth it?” framing.
