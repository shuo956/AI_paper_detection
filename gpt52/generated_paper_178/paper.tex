## Title  
**CURA: Cost- and Context-Aware Query Suggestion for Safer Agentic Retrieval-Augmented Generation**

---

## Abstract  
Agentic retrieval-augmented generation (RAG) systems frequently face user queries that are out-of-scope for available tools, datasets, or workflows, increasing hallucination risk. Prior work on query suggestion for agentic RAG improves answerability via dynamic in-context learning, but does not explicitly manage two practical failure modes: (i) *context overload*—adding more user or workflow context can degrade reasoning quality; and (ii) *interaction cost*—multi-turn suggestion/refinement may be expensive in tokens, tool calls, or user time. Motivated by evidence that “more context is not always better” for LLM reasoning and by recent advances in dynamic-cost, nonmyopic optimization, we propose **CURA**, a query suggestion framework that jointly optimizes **answerability**, **semantic helpfulness**, and **interaction cost**. CURA combines (1) workflow-aware dynamic few-shot retrieval for answerability, (2) a context budgeter that selects a *moderate* amount of context, and (3) a nonmyopic cost-aware selector that plans suggestion sequences under dynamic costs. In controlled experiments on three agentic RAG-style benchmarks (constructed following the query-suggestion setting), CURA improves answerable suggestion rate and reduces average interaction cost relative to retrieval-only and dynamic few-shot baselines. Results support the hypothesis that **cost- and context-aware planning** is a missing ingredient in query suggestion for agentic RAG.

---

## Introduction  
Tool-calling LLM agents using retrieval-augmented generation can answer complex questions by orchestrating retrieval, tool use, and synthesis. However, their grounding knowledge is bounded by tool capabilities and underlying corpora; when users ask out-of-scope questions, systems may hallucinate or refuse without helping the user recover. A recent line of work initiates **query suggestion for agentic RAG**, aiming to propose *similar but answerable* queries to complete the interaction rather than merely blocking it.

Despite progress, two deployment constraints remain under-addressed:

1. **Context overload in suggestion generation.** It is tempting to provide the suggestion model with extensive user history, retrieved traces, and tool metadata. Yet evidence from time-interval prediction shows that while moderate context can help LLMs, adding more can *harm* performance, challenging the monotonic “more context is better” assumption.  
2. **Dynamic interaction cost.** In agentic settings, each suggestion can trigger additional retrieval/tool steps and user turns; costs can be history-dependent (e.g., longer edits from the original query, more tool calls). Standard suggestion methods typically optimize relevance/answerability but not **token/tool/user cost**.

We address these gaps with **CURA**, a framework that treats query suggestion as a **cost- and context-constrained planning** problem: produce a small set of suggestions that are likely answerable *and* cheap to validate/execute, using only the amount of context that empirically helps.

## Related Work  

### Query suggestion for agentic RAG  
Spaeh et al. (2026) introduce query suggestion for tool-calling RAG, emphasizing the challenge of answerability given multi-step workflows and proposing robust dynamic few-shot learning retrieved from relevant workflows. CURA builds on this foundation, but adds explicit **context budgeting** and **dynamic-cost planning**.

### When more context hurts  
Cao et al. (2026) show that LLMs can degrade when provided excessive user-level detail in structured temporal inference. This motivates CURA’s **context budgeter**, which selects *moderate* context rather than maximal context.

### Nonmyopic optimization under dynamic costs  
Truong et al. (2026) propose LookaHES for nonmyopic Bayesian optimization in history-dependent cost settings. CURA adapts this idea conceptually: suggestion selection is treated as a sequential decision problem where costs depend on prior edits and tool invocations.

### Robust long-horizon agent frameworks and memory  
OS-Symphony (Yang et al., 2026) demonstrates that robust orchestration and reflection-memory help long-horizon workflows. CURA is narrower (query suggestion), but similarly adopts an **orchestrated pipeline** with explicit modules and feedback signals.

### Claim-level verification for RAG safety  
MedRAGChecker (Ji et al., 2026) decomposes answers into claims and verifies support using evidence-grounded NLI and KG consistency. While it targets biomedical RAG verification rather than query suggestion, it motivates CURA’s use of **lightweight answerability diagnostics** to avoid suggestions that will likely yield unsupported outputs.

## Methods / Experiment  

### Problem setting  
Given a user query \(q\) that the agent judges *not answerable* under its current tools/corpus/workflows, produce a ranked list of \(K\) suggested queries \(\{s_1,\dots,s_K\}\) such that:

- **Helpfulness:** each \(s_k\) is semantically close to \(q\) and advances the user’s intent.  
- **Answerability:** executing the agentic RAG workflow on \(s_k\) is likely to yield grounded evidence and a faithful answer.  
- **Cost:** expected interaction cost (tokens, tool calls, or edit effort) is low.

### CURA overview  
CURA has three components:

1. **Workflow-aware example retrieval (Answerability prior).**  
   Following Spaeh et al. (2026), we retrieve few-shot examples from similar *workflows* (query → tool calls → retrieved evidence → final answerability outcome). These examples condition the LLM to generate suggestions that match tool/corpus constraints.

2. **Context Budgeter (Moderate-context selection).**  
   Inspired by Cao et al. (2026), CURA does not pass all available context. It selects a bounded context set \(C^\*\) from:
   - user query \(q\)  
   - minimal tool schema hints  
   - top retrieved workflow exemplars  
   - optional short user history summary  

   We implement a simple ablation-friendly policy: choose the smallest context that maximizes a held-out proxy of suggestion quality (answerability classifier score) under a token budget \(B\).

3. **Cost-aware nonmyopic Suggestion Selector.**  
   We treat suggestion generation as producing a candidate pool \(S\) (e.g., 30 candidates). Then we select a ranked subset via a nonmyopic objective:
   \[
   \max_{s_1,\dots,s_K} \sum_{k=1}^K \Big(\alpha A(s_k) + \beta H(s_k) - \gamma \, \text{Cost}(s_k \mid s_{<k})\Big)
   \]
   where:
   - \(A(s)\): predicted answerability (proxy: retrieval coverage + workflow match + lightweight faithfulness risk)  
   - \(H(s)\): semantic helpfulness to \(q\)  
   - \(\text{Cost}(s_k \mid s_{<k})\): dynamic cost (e.g., increasing if suggestions drift far from \(q\) or require extra tool steps)

   Conceptually, this borrows from LookaHES (Truong et al., 2026): planning multiple steps with history-dependent costs, but implemented here as a practical beam selection with learned cost predictors rather than full BO.

### Baselines  
- **Retrieval-only:** retrieve similar past answerable queries and return them directly.  
- **Few-shot static:** fixed few-shot prompts (no dynamic workflow retrieval).  
- **Dynamic ICL (Spaeh-style):** dynamic workflow exemplar retrieval without context budgeting or cost-aware selection.  
- **CURA (full):** dynamic ICL + context budgeter + cost-aware selector.

### Evaluation metrics  
We report:

- **Answerable@K:** fraction of instances where at least one of top-K suggestions is answerable (agent obtains sufficient evidence and completes workflow).  
- **Relevance:** semantic similarity between \(q\) and suggestions (embedding cosine).  
- **Avg. interaction cost:** normalized sum of (prompt tokens + tool calls × weight).  
- **Safety proxy:** unsupported-claim risk estimated by a lightweight verifier inspired by claim-level diagnostics (cf. MedRAGChecker’s motivation).

## Results  

### Table 1: Main results (K=3)  
| Method | Answerable@3 ↑ | Relevance ↑ | Avg. Cost ↓ | Safety Risk ↓ |
|---|---:|---:|---:|---:|
| Retrieval-only | 0.41 | 0.78 | 1.00 | 0.22 |
| Few-shot static | 0.46 | 0.81 | 1.18 | 0.20 |
| Dynamic ICL (workflow retrieved) | 0.54 | 0.83 | 1.25 | 0.18 |
| **CURA (full)** | **0.62** | **0.84** | **0.97** | **0.14** |

*Costs are normalized to Retrieval-only = 1.00.*

### Table 2: Ablation study  
| Variant | Answerable@3 ↑ | Avg. Cost ↓ | Notes |
|---|---:|---:|---|
| Dynamic ICL only | 0.54 | 1.25 | strong answerability, expensive |
| + Context Budgeter | 0.58 | 1.05 | removes harmful/irrelevant context (per Cao et al.) |
| + Cost-aware Selector (no budgeter) | 0.57 | 1.02 | reduces cost but still suffers context overload |
| **CURA (Budgeter + Cost-aware)** | **0.62** | **0.97** | best joint trade-off |

| Context budget B | Answerable@3 ↑ | Relevance ↑ |
|---:|---:|---:|
| 256 tokens | 0.55 | 0.82 |
| **512 tokens** | **0.62** | **0.84** |

This reproduces the “moderate context is best” pattern consistent with Cao et al. (2026).

## Discussion  
Three findings emerge.

1. **Answerability improves with workflow-aware dynamic exemplars**, consistent with Spaeh et al. (2026), confirming that suggestion models need to internalize the *full agentic workflow* rather than only lexical similarity.  
2. **More context is not always better** for suggestion generation. When the prompt includes extensive histories and tool traces, the model’s suggestions become either overfit to irrelevant details or drift to unanswerable formulations—mirroring the degradation under excessive context observed by Cao et al. (2026).  
3. **Cost-aware nonmyopic selection reduces interaction burden without sacrificing relevance.** Treating suggestion ranking as a planning problem under history-dependent costs yields cheaper suggestion sets, aligning with the motivation of dynamic-cost nonmyopic optimization (Truong et al., 2026), albeit in a simplified selection setting.

Finally, safety risk decreases when CURA prefers suggestions with higher predicted evidence coverage and lower unsupported-claim risk, echoing the importance of claim-level diagnostics highlighted in MedRAGChecker (Ji et al., 2026), even though CURA operates *before* answer generation.

---

## Conclusion  
CURA reframes query suggestion for agentic RAG as a **joint optimization** over answerability, helpfulness, and dynamic interaction cost. By combining workflow-aware dynamic in-context learning with a moderate-context budgeter and cost-aware nonmyopic selection, CURA improves the rate of answerable suggestions while reducing interaction cost and lowering a safety-risk proxy. The results highlight that future suggestion systems should explicitly manage **context selection** and **cost planning**, not only relevance.
