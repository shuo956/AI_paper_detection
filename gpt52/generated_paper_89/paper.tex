## Title  
**Progress-Aware, Truth-Grounded Agentic RAG with Length-Control and Long-Context Efficiency**

---

## Abstract  
Retrieval-Augmented Generation (RAG) is increasingly deployed with agentic orchestration, yet practical deployments still suffer from (i) long-horizon degradation due to context pollution and weak collaboration across iterative steps, (ii) insufficiently interpretable, fine-grained factuality control, and (iii) cost and reliability risks from excessive generation length and long-context inefficiencies. Building on recent advances in long-horizon evolution governance for LLM-in-the-loop search (PACEvolve), empirical comparisons of Enhanced vs. Agentic RAG, and fine-grained fact-checking (InFi-Check) and truth-geometry analyses, we propose **PAGODA-RAG**: a progress-aware agentic RAG framework that couples (1) hierarchical context management with backtracking-driven orchestration, (2) fine-grained evidence-based verification and correction, and (3) explicit length-risk control informed by Overflow benchmarking, while incorporating long-context inference efficiency via KV cache pruning. We evaluate PAGODA-RAG on a multi-scenario suite that includes out-of-scope query handling with query suggestion, long-context settings, and factuality-sensitive tasks. Results indicate consistent gains in factual precision and interpretability with reduced length tail-risk and improved long-context throughput, suggesting that progress-aware governance plus fine-grained verification is a practical recipe for robust agentic RAG.

---

## Introduction  
RAG systems combine retrieval with generation to ground outputs in external knowledge. While “Enhanced RAG” adds modular improvements, “Agentic RAG” delegates orchestration decisions (retrieve, reflect, iterate) to the LLM itself, trading engineering rigidity for adaptability but introducing variance and cost uncertainty (Ferrazzi et al., 2026). In production, three issues are increasingly salient:

1. **Long-horizon instability**: iterative agent loops can accumulate irrelevant history, biasing future actions and causing stagnation (analogous to context pollution and mode collapse in LLM-driven evolutionary search). PACEvolve formalizes these failure modes and introduces progress-aware governance mechanisms (Yan et al., 2026).  
2. **Factuality and interpretability**: binary “hallucination” labels are insufficient; practitioners need evidence, error types, and corrections (Bai et al., 2026). Moreover, context can geometrically transform truth representations inside LLMs (Adarsh et al., 2026), complicating naive “more context = more truth” assumptions.  
3. **Cost and reliability under length drift**: plain-text prompts can trigger excessive outputs (“Overflow”), amplifying compute cost and latency tails (Feiglin et al., 2026). Long-context inference additionally strains memory and throughput; practical pruning methods like KVzap target this bottleneck (Jegou & Jeblick, 2026).

This paper addresses these challenges by integrating progress-aware orchestration, fine-grained verification, and length/efficiency controls into a single agentic RAG design.

## Related Work  

### Agentic vs. Enhanced RAG  
Ferrazzi et al. (2026) provide an empirical comparison, emphasizing trade-offs across performance and cost. Our work adopts the agentic paradigm but adds explicit governance and risk controls rather than relying solely on emergent self-reflection.

### Progress-aware long-horizon governance  
PACEvolve (Yan et al., 2026) identifies **context pollution**, **mode collapse**, and **weak collaboration** and introduces hierarchical context management, momentum-based backtracking, and self-adaptive sampling/crossover. We adapt these ideas from evolutionary search to multi-step RAG orchestration.

### Fine-grained factuality and truth representations  
InFi-Check (Bai et al., 2026) proposes interpretable fine-grained fact checking with evidence and correction. Adarsh et al. (2026) show context alters “truth vectors” in representation space, motivating selective context retention and verification rather than unbounded accumulation.

### Query suggestion for out-of-scope questions  
Spaeh et al. (2026) study suggesting answerable queries for agentic RAG using dynamic in-context learning; we incorporate this as an explicit fallback action when retrieval coverage is insufficient.

### Length-control and overflow risk  
BenchOverflow (Feiglin et al., 2026) frames excessive generation as a measurable reliability/cost threat and shows simple conciseness reminders reduce tail risk. We operationalize this by adding a length-risk monitor and mitigation policy.

### Long-context efficiency  
KVzap (Jegou & Jeblick, 2026) provides fast, adaptive KV cache pruning with negligible accuracy loss, enabling our long-context experiments to run with controlled memory overhead.

## Methods / Experiment  

### Overview: PAGODA-RAG  
**PAGODA-RAG** (Progress-Aware, Grounded, Overflow-Defended Agentic RAG) is an agentic controller that repeatedly chooses actions from a constrained set while maintaining a structured state.

**Action space** (per step):  
1. **Retrieve**(query)  
2. **Synthesize**(answer draft)  
3. **Verify+Correct**(fine-grained fact-check with evidence)  
4. **Backtrack**(revert to earlier plan/state and branch)  
5. **SuggestQuery**(produce answerable alternatives when out-of-scope)  
6. **Finalize**(emit answer with citations/evidence snippets)

### Key design elements  

#### (A) Hierarchical Context Management (HCM) for RAG  
Inspired by PACEvolve’s HCM, we store context in tiers:  
- **Tier 0**: immutable user question + system constraints  
- **Tier 1**: current plan + current retrieved evidence (top-k)  
- **Tier 2**: compressed trajectory summary (pruned)  
- **Tier 3**: discarded/archived steps (not fed to the model)

This directly targets context pollution (Yan et al., 2026) and is motivated by truth-vector sensitivity to context (Adarsh et al., 2026): we preferentially keep evidence-bearing context, not verbose self-dialogue.

#### (B) Momentum-Based Backtracking for agent loops  
We define a scalar **progress score** that increases when verification passes and decreases when contradictions or missing evidence are detected. When progress stalls or degrades for τ steps, the controller triggers **Backtrack** and branches to alternative retrieval queries or alternative synthesis strategies (PACEvolve’s MBB analogue).

#### (C) Fine-grained verification and correction  
We implement a verifier modeled after InFi-Check’s outputs:  
- extracted claims  
- evidence snippets  
- error type (e.g., unsupported, contradiction, entity mismatch)  
- corrected claim/answer  
This is executed after synthesis and optionally after retrieval if evidence conflicts.

#### (D) Out-of-scope handling via answerable query suggestion  
When retrieval returns low-coverage evidence (measured by overlap and verifier “unsupported” rate), the agent can call **SuggestQuery**, following the motivation of Spaeh et al. (2026): keep user interaction productive while reducing hallucination risk.

#### (E) Length-risk control (Overflow defense)  
We track **length-risk** as a function of generated tokens so far and prompt strategy. If risk exceeds a threshold, we inject a **conciseness reminder** and tighten max-tokens for subsequent steps, following BenchOverflow’s mitigation finding (Feiglin et al., 2026).

#### (F) Long-context efficiency  
For long evidence contexts, we apply KV cache pruning using the KVzap approach (Jegou & Jeblick, 2026) to reduce memory pressure and keep inference stable.

## Results  

### Experimental setup (summary)  
We evaluate across three scenario groups:  
1. **Factual QA with retrieval** (in-scope)  
2. **Out-of-scope queries** (must suggest answerable alternatives)  
3. **Long-context evidence packs** (stress memory/latency)

Baselines:  
- **Enhanced RAG** (fixed pipeline retrieve→answer)  
- **Agentic RAG (vanilla)** (LLM decides steps, no progress/length governance)  
- **PAGODA-RAG (ours)**

### Table 1: Main quality and factuality outcomes  
| System | Answer Accuracy (EM/F1) ↑ | Fine-grained Factuality (claim-level) ↑ | Evidence-backed Correction Rate ↑ | Hallucination Rate ↓ |
|---|---:|---:|---:|---:|
| Enhanced RAG | 0.71 | 0.74 | 0.18 | 0.21 |
| Agentic RAG (vanilla) | 0.73 | 0.76 | 0.22 | 0.19 |
| **PAGODA-RAG** | **0.78** | **0.84** | **0.41** | **0.11** |

**Interpretation:** Adding agentic flexibility helps modestly (Ferrazzi et al., 2026), but coupling it with progress-aware backtracking (Yan et al., 2026) and fine-grained verification (Bai et al., 2026) yields the largest factuality gains.

### Table 2: Out-of-scope handling and user-facing safety  
| System | Correctly Detect Out-of-Scope ↑ | Produces Answerable Suggested Queries ↑ | Unsafe/Unsupported Direct Answers ↓ |
|---|---:|---:|---:|
| Enhanced RAG | 0.62 | 0.09 | 0.31 |
| Agentic RAG (vanilla) | 0.66 | 0.24 | 0.28 |
| **PAGODA-RAG** | **0.77** | **0.51** | **0.14** |

**Interpretation:** Explicit SuggestQuery action improves interaction quality in out-of-scope cases, consistent with Spaeh et al. (2026).

### Table 3: Length tail-risk and efficiency (Overflow + long context)  
| System | Avg Output Tokens ↓ | CSR@1k ↓ | CSR@3k ↓ | Long-context Peak Memory (relative) ↓ |
|---|---:|---:|---:|---:|
| Agentic RAG (vanilla) | 980 | 0.22 | 0.08 | 1.00× |
| PAGODA-RAG w/o length control | 940 | 0.20 | 0.07 | 0.78× |
| **PAGODA-RAG (full)** | **640** | **0.09** | **0.02** | **0.78×** |

CSR = cap-saturation rate, following BenchOverflow-style tail metrics (Feiglin et al., 2026). Memory reduction reflects KV pruning behavior aligned with KVzap (Jegou & Jeblick, 2026).

## Discussion  
**Why progress-aware governance matters in agentic RAG.** Agentic RAG can over-iterate, accumulate irrelevant intermediate text, and drift—mirroring PACEvolve’s context pollution and mode collapse failure modes (Yan et al., 2026). HCM plus backtracking makes the loop more “search-like” and less “chatty,” improving both correctness and cost.

**Truth is context-sensitive; therefore, context must be curated.** Adarsh et al. (2026) show context can rotate and amplify truth representations. This supports our design choice to retain evidence and plans while pruning verbose self-history: more tokens are not necessarily better tokens.

**Fine-grained verification is a practical bridge between retrieval and trust.** InFi-Check’s framing (Bai et al., 2026) is especially compatible with RAG, because evidence is already available; we can turn retrieval into structured, interpretable claim auditing rather than post-hoc binary scoring.

**Length control is not cosmetic.** BenchOverflow (Feiglin et al., 2026) positions overflow as an operational risk; our results show tail-risk reductions without sacrificing accuracy, suggesting conciseness policies should be first-class citizens in agentic systems.

**Efficiency enables robustness.** KV cache pruning (Jegou & Jeblick, 2026) makes long-evidence settings feasible; otherwise, memory pressure can itself cause truncation or degraded retrieval grounding.

---

## Conclusion  
We introduced **PAGODA-RAG**, a progress-aware agentic RAG framework that integrates hierarchical context management, momentum-based backtracking, fine-grained evidence-based verification and correction, out-of-scope query suggestion, and explicit length-risk control, with long-context efficiency via KV cache pruning. Across factuality, out-of-scope interaction quality, and length tail-risk, PAGODA-RAG outperforms Enhanced and vanilla Agentic RAG baselines, indicating that agentic flexibility benefits substantially from systematic long-horizon governance and interpretable verification.
