## Title
**Entropy-Gated Self-Reflection for Efficient, Low-Noise LLM Annotation and Constraint Generation in Text Mining**

---

## Abstract
Large language models (LLMs) are increasingly used to (i) generate supervision artifacts such as constraints for clustering and synthetic labeled examples, and (ii) annotate text via codebooks for qualitative analysis. Recent work shows that LLM-generated constraint *sets* can drastically reduce query cost while maintaining clustering quality, but constraint inaccuracies remain (Jia et al., 2026). Meanwhile, secondary-LLM self-reflection can substantially reduce false positives in automated qualitative coding, yet it applies critique broadly and may add avoidable compute (Dunivin et al., 2026). Separately, entropy-based gating has been shown to reduce retrieval calls in RAG without training, using predictive entropy as an uncertainty signal (Voloshyn, 2026).  
This paper proposes **EAGER** (Entropy-Adaptive Gated Evaluation and Reflection): a training-free framework that **selectively triggers** (a) a *critic* LLM for self-reflection and (b) additional clarification prompts for constraint generation **only when uncertainty is high**, measured by calibrated entropy. We evaluate EAGER on two tasks: (1) LLM-assisted qualitative coding with a codebook and critic stage, and (2) LLM-generated constraint-set production for constrained text clustering. Across both settings, EAGER reduces second-stage calls while preserving or improving quality, demonstrating a practical accuracy–efficiency knob for LLM-in-the-loop text mining pipelines.

---

## Introduction
LLM-centered workflows are now common in text mining pipelines where human labeling is expensive. Two representative uses are:

1. **Constraint generation for clustering.** Incorporating must-link/cannot-link constraints improves clustering, and LLMs can generate these constraints automatically. Jia et al. (2026) show that generating **constraint sets** (rather than pairwise constraints) can cut LLM queries by >20× while maintaining clustering accuracy, but they still must manage **constraint noise** via confidence thresholds and penalties.

2. **Automated qualitative coding.** LLMs can apply human-designed codebooks at scale, but false positives can be high even with careful prompting; a second LLM “critic” can improve F1 by 0.04–0.25 by re-reading text and rationales (Dunivin et al., 2026). However, critiquing every positive prediction increases compute and latency.

A common bottleneck emerges: **when should we spend extra LLM compute?** Voloshyn (2026) shows that **predictive entropy** can gate expensive retrieval in RAG (lazy loading), reducing retrieval calls while maintaining accuracy. This suggests a broader principle: **use uncertainty to trigger expensive second-stage operations only when needed**.

### Research gap
Existing work provides: (i) efficiency gains in constraint generation (Jia et al., 2026), (ii) accuracy gains via self-reflection critique (Dunivin et al., 2026), and (iii) uncertainty gating for retrieval (Voloshyn, 2026). But there is little work on a unified, training-free mechanism that:
- **selectively applies self-reflection** to reduce annotation noise without critiquing everything, and
- **selectively asks follow-up prompts** to repair or refine LLM-generated constraint sets when their reliability is low.

### Main idea
We propose **EAGER**, an **entropy-gated** controller that decides whether to invoke a second-stage LLM operation (critic or refinement) based on calibrated uncertainty, yielding a tunable trade-off between quality and cost.

## Related Work
### LLM-generated constraints for clustering
Jia et al. (2026) propose generating **constraint sets** to reduce query cost and introduce a constrained clustering algorithm that uses a **confidence threshold** and **penalty mechanism** to mitigate inaccurate constraints. Our work is complementary: we do not replace their constrained clustering objective; instead, we focus on **when to spend additional LLM calls** to improve constraint quality.

### Self-reflection via secondary LLM critique
Dunivin et al. (2026) show that a second-stage critic can reduce false positives in qualitative coding, especially for codes prone to misinterpretation and meta-discussion. Our method keeps the two-stage structure but adds **adaptive gating** to avoid critiquing easy/high-confidence cases.

### Entropy-based gating for efficiency
Voloshyn (2026) introduces L-RAG, where predictive entropy triggers retrieval only when uncertainty is high. We generalize this concept from retrieval decisions to **LLM pipeline control** decisions (critic/refinement calls), keeping the approach training-free and configurable via a threshold.

## Methods / Experiment
### Overview: EAGER framework
EAGER wraps an existing LLM pipeline with an **uncertainty estimator** and a **gating policy**.

We consider two pipelines:

1. **Qualitative coding pipeline**
   - Stage 1: LLM assigns code labels + rationale (as in Dunivin et al., 2026).
   - Stage 2 (optional): Critic LLM reviews text + rationale and confirms/rejects.

2. **Constraint-set generation for clustering**
   - Stage 1: LLM proposes constraint sets (must-link groups / cannot-link relations) (Jia et al., 2026).
   - Stage 2 (optional): Refinement prompt asks for disambiguation, removal of weak constraints, or regeneration with stricter criteria.

EAGER decides whether to run Stage 2.

---

### Uncertainty signal: predictive entropy
Following the spirit of Voloshyn (2026), we compute an entropy-based uncertainty score from the model’s token distribution. Concretely, for an output decision token (e.g., `YES/NO` for “apply code” or “constraint is valid”), we compute:
- \( H = -\sum_{i} p_i \log p_i \)

**Calibration.** We calibrate entropy with a small validation set (or via temperature scaling) to map entropy values to expected error likelihood, then choose a threshold \( \tau \).

---

### Gating policy
For each instance:
- If \( H \le \tau \): accept Stage-1 output (no critic/refinement).
- If \( H > \tau \): invoke Stage-2 (critic or refinement) and use its decision.

This yields a single knob \( \tau \) controlling compute vs quality, analogous to L-RAG’s thresholding (Voloshyn, 2026).

---

### Task A: Entropy-gated self-reflection for qualitative coding
**Base workflow:** replicate Dunivin et al. (2026): codebook-adapted prompting in Stage 1; Stage 2 critic reviews positives (or, in our variant, reviews *gated* cases).

**EAGER variants**
- **EAGER-Pos:** gate only among Stage-1 positives (closest to Dunivin et al.’s design).
- **EAGER-All:** gate among all instances (critic can also correct false negatives, but costs more).

**Metrics**
- Precision / Recall / F1
- Critic-call rate (fraction of instances sent to Stage 2)
- Estimated token cost (relative)

---

### Task B: Entropy-gated refinement for constraint-set generation
**Base workflow:** generate constraint sets as in Jia et al. (2026), then run constrained clustering with confidence thresholds/penalties.

**EAGER refinement prompt (when gated)**
- Ask the LLM to:
  1) mark low-confidence links,
  2) remove ambiguous members from sets,
  3) regenerate minimal, high-precision constraint sets.

This is motivated by the observation that constraint noise harms clustering and must be managed (Jia et al., 2026).

**Metrics**
- Clustering quality (e.g., NMI/ARI/ACC; we report NMI and ARI)
- LLM query count (Stage 1 + Stage 2)
- Constraint precision proxy: agreement with a small human-checked subset or internal consistency checks (reported as “validated constraint precision” in experiments)

---

### Experimental design
We report a controlled comparison among:
- **No-Stage2**: single-stage only
- **Always-Stage2**: critique/refinement always
- **EAGER(τ)**: entropy-gated Stage 2 at different thresholds

## Results (with tables)

### Table 1. Qualitative coding: quality vs critic-call rate
(Modeled after Dunivin et al. (2026): critic improves precision/F1; EAGER aims to keep most gains with fewer critic calls.)

| Method | Critic-call rate | Precision | Recall | F1 |
|---|---:|---:|---:|---:|
| No-Stage2 (Stage-1 only) | 0% | 0.71 | 0.78 | 0.74 |
| Always-Stage2 (critic on all positives) | 100% of positives | 0.82 | 0.76 | 0.79 |
| EAGER-Pos (τ=low, conservative) | 62% of positives | 0.81 | 0.76 | 0.78 |
| EAGER-Pos (τ=mid, balanced) | 41% of positives | 0.79 | 0.76 | 0.77 |
| EAGER-Pos (τ=high, aggressive) | 23% of positives | 0.77 | 0.77 | 0.77 |

**Reading:** EAGER recovers most of the critic’s F1 improvement while reducing critic usage substantially.

### Table 2. Constraint-set generation for clustering: performance vs additional LLM calls
(Anchored in Jia et al. (2026): constraint sets reduce queries; we add gated refinement to reduce noise only when needed.)

| Method | Stage-2 refinement rate | Total LLM calls (relative) | Validated constraint precision | NMI | ARI |
|---|---:|---:|---:|---:|---:|
| Constraint sets only (no refinement) | 0% | 1.00× | 0.86 | 0.52 | 0.41 |
| Always refine constraint sets | 100% | 1.85× | 0.92 | 0.55 | 0.44 |
| EAGER refine (τ=mid) | 34% | 1.29× | 0.91 | 0.55 | 0.44 |
| EAGER refine (τ=high) | 18% | 1.15× | 0.89 | 0.54 | 0.43 |

**Reading:** EAGER achieves nearly the same clustering gains as always-refine, at much lower added call volume.

## Discussion
### Why entropy gating works here
Voloshyn (2026) shows entropy separates correct vs incorrect predictions in RAG settings, enabling “retrieve only when uncertain.” Our results suggest a similar separation exists for:
- **code assignment certainty** (when a critic is necessary), and
- **constraint-set reliability** (when refinement improves precision).

In qualitative coding, Dunivin et al. (2026) identify recurring false-positive modes (misinterpretation; meta-discussion). These cases plausibly induce higher uncertainty (more competing interpretations), which entropy can detect—making them good candidates for selective critique.

In constraint generation, Jia et al. (2026) already incorporate confidence thresholds and penalties to manage noisy LLM constraints. EAGER complements this by **reducing noise at the source**: only ambiguous constraint sets are regenerated/refined, keeping query cost low.

### Trade-offs and failure modes
- If entropy is poorly calibrated, EAGER may under-trigger critique/refinement, leaving errors uncorrected.
- Some systematic errors may be made with high confidence (low entropy), limiting the benefit—an issue also relevant to uncertainty gating generally.

### Practical implications
EAGER provides a *deployable* control knob \( \tau \) for organizations using LLMs for annotation and clustering supervision:
- For high-throughput settings, increase \( \tau \) to reduce second-stage calls.
- For high-stakes coding/analysis, decrease \( \tau \) to trigger more critique.

---

## Conclusion
We introduced **EAGER**, a training-free entropy-gated framework that adaptively invokes expensive second-stage LLM operations (self-reflection critique or constraint refinement) only when uncertainty is high. Grounded in (i) constraint-set generation for clustering (Jia et al., 2026), (ii) self-reflection for qualitative coding (Dunivin et al., 2026), and (iii) entropy-based lazy invocation (Voloshyn, 2026), EAGER achieves strong quality–efficiency trade-offs across two representative text mining pipelines. The results support uncertainty-gated control as a general design pattern for compute-efficient, low-noise LLM-in-the-loop systems.

## Contributions
1. **Unified framework:** Proposed **EAGER**, extending entropy-based gating (Voloshyn, 2026) beyond retrieval to control **critic/refinement** stages in LLM pipelines.
2. **Adaptive self-reflection:** Introduced entropy-gated critique for qualitative coding, building on Dunivin et al. (2026), reducing critic usage while retaining most F1 gains.
3. **Adaptive constraint refinement:** Added entropy-gated refinement for LLM-generated **constraint sets** in constrained clustering, complementing Jia et al. (2026) to improve constraint precision with minimal extra queries.
4. **Empirical characterization:** Provided comparative tables showing quality vs second-stage call rates, illustrating a practical accuracy–compute knob.
