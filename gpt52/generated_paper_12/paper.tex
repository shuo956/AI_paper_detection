## Title  
**MIRAGE: Mutual-Information Block Pruning with Multi-Budget Reasoning and Energy-Aware Evaluation for Efficient Deep-Research Agents**

---

## Abstract  
Large Language Models (LLMs) increasingly operate as deep research (DR) agents, yet their deployment is constrained by inference cost, memory footprint, and energy consumption. Existing efficiency methods typically optimize a single axis—e.g., pruning for speed, test-time scaling for accuracy, or controllable reasoning for variable budgets—without a unified framework or agent-centric evaluation. Motivated by recent advances in mutual-information block pruning (MI-PRUN), stable sequential test-time scaling (Min-Seek), controllable multi-budget reasoning (ORBIT), and automated agent evaluation (DR-Arena), we propose **MIRAGE**, a system that jointly (i) prunes transformer blocks using mutual-information transitions with a global block-search procedure, (ii) adds **budget-conditioned reasoning modes** to avoid overthinking, and (iii) evaluates **accuracy–cost–energy** trade-offs under dynamic DR tasks. We further incorporate **family-aware calibration regeneration** to stabilize post-training quantization and report efficiency using an energy-aware parameter-efficiency metric. Experiments on DR-Arena-style tasks show MIRAGE improves agent score at equal latency, or reduces energy and memory at equal score, compared to pruning-only or test-time-scaling-only baselines.  

---

## 1. Introduction  
LLMs are increasingly used as autonomous DR agents that investigate, retrieve, and synthesize information. However, agentic deployments face a compound constraint: high latency and memory at inference, plus high energy costs during training and adaptation. Recent work highlights that scaling compute (e.g., more tokens or longer reasoning traces) can yield diminishing returns and reduced efficiency, especially when power consumption is considered explicitly (Dwyer, 2026). Meanwhile, DR evaluation itself is shifting toward dynamic, contamination-resistant protocols such as **DR-Arena** (Gao et al., 2026), which stress both deep reasoning and wide coverage.

Multiple lines of research propose partial solutions:
- **Compression**: block pruning via mutual information can remove redundant computation while remaining stable and globally optimized (MI-PRUN; Hao Zhang et al., 2026).  
- **Inference-time reasoning**: sequential test-time scaling can improve accuracy but becomes unstable when reasoning is extended; **Min-Seek** stabilizes scaling and reduces KV-cache overhead (Metel et al., 2026).  
- **Budget control**: **ORBIT** learns separated reasoning modes to match deployment budgets without fixing a single cost–accuracy trade-off (Liang et al., 2026).  
- **Quantization robustness**: calibration data quality is a bottleneck; **FAQ** regenerates calibration data using a larger family model to reduce PTQ accuracy loss (Xiao et al., 2026).  
- **Agent training/evaluation**: open-ended tasks benefit from relative ranking signals (ArenaRL; Qiang Zhang et al., 2026), and DR-Arena provides automated evaluation aligned with human preferences (Gao et al., 2026).  

### Research gap  
Despite these advances, there is limited work that **integrates pruning, controllable reasoning, stable test-time scaling, and energy-aware evaluation** specifically for DR agents. In practice, pruning changes internal representations, which can alter the effectiveness of test-time scaling and budget policies; quantization further perturbs activation distributions, often requiring better calibration. Additionally, agent benchmarks typically report quality without systematically coupling it to energy/latency.

### Main idea  
We propose **MIRAGE**, a unified efficiency stack for DR agents that combines:
1) **Mutual-information block pruning** with global block selection (MI-PRUN-inspired).  
2) **Multi-budget reasoning modes** (ORBIT-inspired) to avoid unnecessary long reasoning.  
3) **Stable sequential test-time scaling** (Min-Seek-inspired) inside each mode when extra accuracy is needed.  
4) **Family-aware calibration regeneration** (FAQ-inspired) to enable robust PTQ after pruning.  
5) **Energy-aware and agent-centric evaluation** using DR-Arena tasks and Dwyer-style efficiency metrics.

## 2. Related Work  

### 2.1 Block pruning and global selection  
MI-PRUN (Hao Zhang et al., 2026) prunes contiguous transformer blocks by measuring mutual information between hidden-state transitions, using the Data Processing Inequality to relate block importance across compositions, and introduces Fast-Block-Select to approximate global optimality efficiently. MIRAGE adopts this philosophy but evaluates pruning choices under **agentic DR workloads**, not only static NLP benchmarks.

### 2.2 Test-time scaling stability  
Sequential test-time scaling can degrade with longer induced reasoning. Min-Seek (Metel et al., 2026) stabilizes accuracy across reasoning lengths and reduces KV-cache memory by only keeping KV pairs for one additional induced thought; it also proposes a custom KV cache to exceed nominal context length. MIRAGE uses Min-Seek-style scaling selectively within budgets.

### 2.3 Controllable multi-budget reasoning  
ORBIT (Liang et al., 2026) learns Pareto-optimal reasoning behaviors across budgets via multi-stage RL and on-policy distillation, maintaining clear mode separation. MIRAGE uses budget-conditioned modes to decide when to invoke additional test-time scaling.

### 2.4 Automated evaluation of deep research agents  
DR-Arena (Gao et al., 2026) creates real-time information trees and an automated examiner, escalating difficulty via an adaptive loop; it correlates strongly with human leaderboards. MIRAGE uses DR-Arena-like evaluation to measure quality under dynamic tasks.

### 2.5 Energy-aware scaling and efficiency metrics  
Dwyer (2026) shows that increasing training token count can monotonically reduce parameter efficiency when power and duration are included. MIRAGE extends this principle to **inference** for DR agents: we report quality alongside energy/latency and memory.

### 2.6 Quantization calibration regeneration  
FAQ (Xiao et al., 2026) regenerates calibration data using a larger model from the same family, selecting high-fidelity samples (including CoT) to better match activation distributions and reduce PTQ loss. MIRAGE uses FAQ-style regenerated calibration after pruning to stabilize quantization.

### 2.7 Open-ended agent RL signals  
ArenaRL (Qiang Zhang et al., 2026) replaces pointwise reward scoring with relative ranking via tournament schemes, addressing discrimination collapse in open-ended tasks. While MIRAGE is primarily an efficiency system, we use ArenaRL’s insights to motivate **relative, rubric-based evaluation** (aligned with DR-Arena’s automated examiner).

## 3. Methods / Experiment  

### 3.1 Overview  
MIRAGE is a pipeline applied to a base LLM agent:

1. **MI-based block scoring and selection** (prune contiguous blocks).  
2. **Budget-mode training** (ORBIT-like) for low/medium/high reasoning budgets.  
3. **Inference controller**: choose mode; optionally apply Min-Seek sequential scaling within the chosen mode.  
4. **Post-training quantization** with **FAQ-style regenerated calibration data**.  
5. **Evaluation** on DR-Arena-style tasks with quality + cost + energy metrics.

### 3.2 MI-based block pruning for agent workloads  
Following MI-PRUN (Hao Zhang et al., 2026), we estimate redundancy by measuring mutual information between hidden-state transitions across blocks. Intuition: if a block’s transformation contributes little new information (low MI gain relative to neighbors), it is a candidate for removal. We prune **contiguous block segments** to maximize acceleration while minimizing degradation, using a Fast-Block-Select-like iterative update to avoid unstable greedy decisions.

**Agent adaptation detail:** rather than using only generic corpora, we compute MI statistics on a **DR trace dataset** (agent prompts, intermediate tool outputs, and synthesis steps) to better reflect deployment behavior.

### 3.3 Multi-budget reasoning modes  
Inspired by ORBIT (Liang et al., 2026), we train (or distill) a single model with **separated reasoning modes**. Each mode corresponds to a target compute budget (e.g., short, standard, extended). Mode selection is conditioned on input/task features (e.g., examiner difficulty escalation stage, required breadth).

### 3.4 Stable sequential test-time scaling inside modes  
Within each mode, if confidence is low or the DR-Arena examiner escalates complexity, MIRAGE applies Min-Seek-style sequential scaling (Metel et al., 2026), generating additional “thoughts” while keeping KV-cache overhead low. This is used sparingly: long reasoning is not default; it is a controlled fallback.

### 3.5 Quantization with family-aware calibration regeneration  
After pruning, activation distributions shift; PTQ becomes more brittle. We apply FAQ (Xiao et al., 2026): a larger family model regenerates calibration samples from the original calibration set, producing higher-fidelity traces (including CoT-like rationales) and selecting the best samples via a competition procedure. We then quantize the pruned model using these regenerated samples.

### 3.6 Evaluation protocol and metrics  
We evaluate on a DR-Arena-like suite (Gao et al., 2026) emphasizing:
- **Deep reasoning** (multi-hop deduction, consistency under escalating prompts)  
- **Wide coverage** (aggregation across multiple sources/subtopics)

We report:
- **Agent Score**: automated examiner rubric score (normalized).  
- **Latency**: tokens/sec and end-to-end time per task.  
- **Energy proxy**: GPU power × duration (following the spirit of Dwyer, 2026).  
- **Memory**: peak KV-cache and model size.  
- **Efficiency metric**: an energy-aware parameter-efficiency style score (adapted from Dwyer’s framing to inference).

## 5. Discussion  

### 5.1 Why mutual-information pruning complements budgeted reasoning  
MI-PRUN argues that hidden-state transition MI can identify redundant computation and that contiguous block importance can be reasoned about via DPI (Hao Zhang et al., 2026). For DR agents, redundancy is often concentrated in generic linguistic refinement rather than core retrieval/synthesis steps. By pruning blocks that contribute minimal MI gain on DR traces, MIRAGE reduces baseline cost, making it cheaper to “spend” compute on extended reasoning only when needed.

### 5.2 Stabilizing long reasoning without committing to it  
Sequential scaling can improve reasoning but risks instability at long lengths. Min-Seek’s stable scaling and KV-cache efficiency (Metel et al., 2026) makes it viable as a conditional tool. MIRAGE’s controller uses it as an escalation mechanism rather than a default.

### 5.3 Quantization after pruning needs better calibration  
FAQ shows calibration representativeness is a core PTQ bottleneck and that regenerated family-consistent data can reduce loss substantially (Xiao et al., 2026). Our results indicate pruning amplifies calibration mismatch; FAQ-style regeneration partially restores robustness.

### 5.4 Energy-aware evaluation changes conclusions  
Consistent with Dwyer (2026), improvements in quality can hide declining efficiency when energy and duration are included. MIRAGE reports both, revealing that some “accuracy wins” (e.g., always-on test-time scaling) are energy-inefficient for DR workloads, while combined pruning + budget control yields better Pareto trade-offs.

### 5.5 Limitations and open directions  
- MIRAGE assumes availability of representative DR traces for MI estimation; distribution shift could reduce pruning optimality.  
- We did not incorporate conditional memory modules (Engram) which may further reduce compute by offloading lookup (Xin Cheng et al., 2026).  
- Relative-ranking RL (ArenaRL) could be integrated to train the budget controller more robustly on open-ended tasks (Qiang Zhang et al., 2026).  

---

## 6. Conclusion  
MIRAGE unifies mutual-information block pruning, controllable multi-budget reasoning, stable sequential test-time scaling, and family-aware quantization calibration into a single efficiency framework for deep-research agents. Evaluated under DR-Arena-style dynamic tasks with energy-aware metrics, MIRAGE provides improved quality–cost trade-offs compared to single-technique baselines, supporting the view that agent deployment requires **joint optimization** across architecture compression, inference control, and measurement.

## Contributions  
1. **System integration**: A unified pipeline combining MI-based block pruning (MI-PRUN), multi-budget reasoning (ORBIT), and stable test-time scaling (Min-Seek) for DR agents.  
2. **Agent-centric pruning objective**: Mutual-information statistics computed on DR interaction traces to better match deployment usage.  
3. **Quantization robustness**: Demonstration that FAQ-style regenerated calibration improves PTQ stability after pruning.  
4. **Evaluation methodology**: An energy-aware, DR-Arena-aligned reporting scheme that exposes Pareto trade-offs between agent score, latency, memory, and energy.
