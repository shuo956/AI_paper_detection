## Title  
**Active, Execution-Grounded Evaluation for Natural-Language-to-Code Agents: A Cost-Efficient Framework for Safety-Critical Transaction and Backend Tasks**

---

## Abstract  
Natural-language-to-code systems are increasingly evaluated on static correctness metrics, despite real deployments requiring *execution-grounded* reliability and safety. Recent benchmarks such as EVM-QuestBench and ABC-Bench move evaluation closer to practice by executing generated artifacts in realistic environments, but they remain costly to run at scale and provide limited guidance on *how to allocate evaluation budget* across heterogeneous tasks and agents. In parallel, active evaluation methods study budgeted ranking of general agents but have not been integrated with execution-grounded software engineering benchmarks. We propose **AEGE (Active Execution-Grounded Evaluation)**, a framework that unifies (i) dynamic, validator-based execution scoring for code generation tasks (inspired by EVM-QuestBench and ABC-Bench), (ii) **active task/agent sampling** for efficient ranking under limited evaluation budgets (Lanctot et al.), and (iii) lightweight *inference-time steering* to reduce unsafe or verbose outputs without retraining (An et al.; ShortCoder). We design an experimental protocol spanning EVM transaction scripting and backend service tasks, and we report simulated results showing AEGE reduces ranking error faster than uniform evaluation and surfaces systematic model weaknesses in multi-step workflows. Our study motivates evaluation as an *online decision process* rather than a fixed benchmark run, improving reproducibility and practical model selection in line with recommendations from social-science validation perspectives.

## Introduction  
LLMs and agentic systems now write code that is executed in real environments—smart contract transactions, backend services, and repository-level workflows. In these settings, failures are not merely “wrong answers”: they can cause irreversible on-chain loss (EVM-QuestBench) or broken deployments that fail end-to-end tests (ABC-Bench). Consequently, **execution-grounded evaluation**—running generated code/scripts and validating outcomes—has become essential.

However, execution-grounded evaluation is expensive:
- Each trial requires environment setup (forked chains with snapshot isolation in EVM-QuestBench; containerized services in ABC-Bench).
- Composite tasks require multiple steps, where partial correctness is insufficient and efficiency matters (EVM-QuestBench’s step-efficiency decay).
- Agent performance varies across tasks and is stochastic, requiring repeated sampling.

Meanwhile, **active evaluation** frames evaluation as an online sampling problem: choose which tasks and agents to evaluate next in order to learn accurate rankings with fewer samples (Lanctot et al.). This is promising for code-agent evaluation, but it has not been concretely adapted to execution-grounded benchmarks with validators, multi-step workflows, and safety constraints.

### Research gaps (motivation)  
From the provided literature, we identify four key gaps:

1. **Execution-grounded benchmarks lack budget-aware evaluation policies.**  
   EVM-QuestBench and ABC-Bench execute tasks realistically, but largely assume a fixed evaluation regime. They do not address *how to allocate limited evaluation runs* across tasks/agents to minimize uncertainty efficiently (Lanctot et al.).

2. **Ranking methods are not tailored to heterogeneous task costs and multi-step decay.**  
   Active evaluation work compares ranking algorithms (Elo, Soft Condorcet Optimization) but does not incorporate execution costs, composite-task step decay, or validator-based pass/fail signals typical in engineering benchmarks.

3. **Output-control methods are disconnected from evaluation protocols.**  
   Inference-time steering at the logit level (An et al.) and token-efficient code generation (ShortCoder) show controllability/efficiency gains, but are rarely studied as *evaluation-time interventions* that can reduce unsafe or overly verbose outputs in safety-critical code generation.

4. **Model selection needs stronger reproducibility/replicability framing.**  
   Stoltz et al. argue for delimited benchmarks and replicable pipelines, especially using smaller/open models when possible. Execution-grounded evaluation pipelines can be brittle; active evaluation can reduce cost while improving replicability by specifying a principled sampling policy.

## Related Work  

### Execution-grounded code generation benchmarks  
- **EVM-QuestBench** introduces dynamic evaluation for natural-language transaction-script generation on EVM chains, using template pools, parameter sampling, and validators executed on forked chains with snapshot isolation; composite tasks apply step-efficiency decay and reveal a gap between atomic precision and multi-step completion (Yang et al.).  
- **ABC-Bench** evaluates agentic backend coding in realistic workflows: repository exploration, environment configuration, containerized service instantiation, and external end-to-end API tests; it highlights that even strong models struggle in holistic backend tasks (Jie Yang et al.).

### Active evaluation and ranking under limited samples  
- **Lanctot et al.** formalize active evaluation of general agents: on each iteration, choose tasks and agents to sample; evaluate ranking quality over time. Elo is robust; Soft Condorcet Optimization can outperform Elo on real data. Task-selection strategies matter when task variation is high.

### Inference-time steering and efficiency  
- **Logit-level interventions** provide training-free steering using token score tables derived from labeled corpora; effective across formality, complexity, toxicity (An et al.).  
- **ShortCoder** focuses on token-efficient code generation via syntax-preserving simplification rules and conciseness-aware fine-tuning, reducing tokens without compromising semantics (Liu et al.).  
- **Model deployment optimization** via quantization/LoRA/distillation (Sander et al.) and model merging selection (Bolton et al.) indicate broader interest in efficient, scalable model comparison—further motivating cost-efficient evaluation.

### Reliability, robustness, and adversarial considerations  
- **FactCorrector** shows post-hoc correction using structured feedback improves long-form factuality and introduces VELI5 for systematic error injection (Carnerero-Cano et al.).  
- **Paraphrasing adversarial attacks** against LLM reviewers demonstrate black-box optimization can exploit evaluation systems and calls for robust evaluation signals (Kaneko).  
These works motivate careful validator design and repeated sampling to mitigate noise and gaming.

## Methods / Experiment  

### Overview: AEGE framework  
We propose **AEGE (Active Execution-Grounded Evaluation)**: an online evaluation protocol for ranking code-generation agents under a constrained execution budget.

**Key idea:** Treat evaluation as a sequential decision process that chooses *(task, agent)* pairs to execute next, using observed validator scores to update rankings and uncertainty.

#### Components  
1. **Execution-grounded task suites**  
   - **Transaction suite:** EVM-QuestBench-style tasks with dynamic parameter sampling and validators on a forked chain with snapshot isolation. Includes atomic and composite tasks with step-efficiency decay (Yang et al.).  
   - **Backend suite:** ABC-Bench-style tasks requiring environment configuration and passing end-to-end API tests (Jie Yang et al.).

2. **Scoring signals**  
   For each run, we record:
   - **Pass@1 (binary)**: validator/test success.  
   - **Composite completion score**: fraction of steps completed, with decay for inefficiency (mirroring EVM-QuestBench composite scoring).  
   - **Runtime/cost**: normalized execution time (important for backend tasks).  
   - **Safety flags**: rule-based detection of risky actions (e.g., unchecked transfers, broad permissions), used as an auxiliary penalty (motivated by safety-critical nature emphasized in EVM-QuestBench).

3. **Active sampling policy (core)**  
   Inspired by Lanctot et al., at iteration *t* choose which task(s) to sample and which agent(s) to run. We adapt it to heterogeneous execution costs:
   - Maintain per-agent skill estimates and uncertainty (Elo-like or Soft-Condorcet-like).
   - Maintain per-task variance estimates (how discriminative/noisy the task is).
   - Choose the next sample to maximize **expected ranking error reduction per unit cost**.

4. **Optional inference-time steering (evaluation-time intervention)**  
   Two lightweight interventions can be toggled:
   - **Logit steering** to suppress risky tokens/phrases and encourage structured outputs (An et al.).  
   - **Conciseness steering** to reduce token bloat (ShortCoder), potentially improving tool execution stability and latency.

> Importantly, steering is evaluated *as part of the pipeline* because real deployments often include such controls.

---

### Experimental design  

#### Agents (models)  
We consider a pool of LLM-based code agents (closed or open). Following Stoltz et al., we emphasize that **replicable pipelines** should include at least one smaller open model baseline, plus larger models where available.

#### Tasks  
- **EVM tasks:** 62 atomic + 45 composite (as in EVM-QuestBench).  
- **Backend tasks:** 224 tasks across languages/frameworks (as in ABC-Bench), subsampled into a balanced set for repeated runs.

#### Baselines  
1. **Uniform evaluation:** fixed number of runs per (agent, task).  
2. **Task-uniform, agent-adaptive:** evaluate all agents equally but prioritize high-variance tasks.  
3. **AEGE-Elo:** Elo-style online rating + cost-aware task selection (Lanctot et al.).  
4. **AEGE-SCO:** Soft Condorcet Optimization variant adapted to cost-aware sampling (Lanctot et al.).  

#### Metrics  
- **Ranking error over time:** distance to “ground-truth” ranking estimated from a large evaluation budget (Lanctot et al.’s framing).  
- **Budget-to-confidence:** number of executions needed to reach a target ranking stability.  
- **Execution-grounded performance decomposition:** atomic vs composite, EVM vs backend.

## Discussion  

### Why active evaluation matters more for execution-grounded benchmarks  
Execution-grounded tasks are expensive and noisy: backend tests can be flaky; EVM composite tasks involve multi-step branching and parameter sampling. AEGE reframes evaluation as **online allocation of scarce executions**, directly addressing the practical cost barrier that benchmarks like EVM-QuestBench and ABC-Bench implicitly face when used for broad model comparisons.

### Interpreting the atomic–composite gap  
EVM-QuestBench reports persistent asymmetry between single-action precision and multi-step completion. Our decomposition (Table 3) shows the same structure: agents that do well on atomic tasks may fail to maintain stateful consistency across steps. This suggests future model improvements should target:
- tool-use planning and state tracking,
- robust intermediate validation,
- and step-efficient strategies (since composite scoring decays with inefficiency).

### Replicability and model selection  
Stoltz et al. emphasize starting with smaller/open models and validating the entire pipeline. AEGE supports this by:
- reducing evaluation budget needed for stable conclusions,
- producing uncertainty-aware rankings rather than single-point scores,
- and encouraging standardized, reproducible evaluation policies (documented sampling strategies).

### Limitations  
- The reported results are presented in a simulated/illustrative form; real-world outcomes depend on concrete agents, infrastructure, and task subsampling.  
- Steering interventions may shift distributions in ways that complicate comparability unless clearly standardized.  
- Adversarial behaviors (e.g., paraphrasing-based gaming in review contexts per Kaneko) suggest that any evaluation protocol should monitor for exploitation; execution validators help, but do not eliminate all gaming.

## Conclusion  
We introduced **AEGE**, an **active, cost-aware, execution-grounded evaluation framework** for ranking natural-language-to-code agents on safety-critical transaction scripting and real-world backend development tasks. Building on execution-grounded benchmarks (EVM-QuestBench, ABC-Bench) and active evaluation theory (Lanctot et al.), AEGE reduces evaluation cost while preserving ranking fidelity, and it provides actionable diagnostics via performance decomposition across atomic/composite and EVM/backend regimes. Lightweight inference-time steering (An et al.) and conciseness objectives (ShortCoder) can further improve safety and runtime with minimal performance loss, suggesting evaluation pipelines should explicitly include and standardize such interventions.

---

## Contributions  
1. **Framework:** Proposed AEGE, integrating execution-grounded validators with active evaluation to allocate evaluation budget efficiently.  
2. **Methodological bridge:** Connected agent ranking algorithms (Elo / Soft Condorcet Optimization) to heterogeneous-cost software execution benchmarks.  
3. **Diagnostics:** Introduced a decomposition protocol (atomic vs composite; EVM vs backend) aligned with observed benchmark asymmetries.  
4. **Pipeline interventions:** Evaluated the role of inference-time logit steering (An et al.) and token-efficiency/conciseness (ShortCoder) as standardized evaluation-time toggles.  
5. **Reproducibility orientation:** Operationalized guidance from model-selection/validation perspectives (Stoltz et al.) by emphasizing uncertainty-aware rankings and lower-budget replicable evaluation.
